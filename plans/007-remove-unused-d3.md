## Why this matters

`packages/rolldown/package.json` declares the umbrella `d3` package (which pulls ~30
sub-packages), but no source file imports `from 'd3'`. The code only uses the granular
`d3-hierarchy` and `d3-shape`. The unused umbrella inflates install time and the
dependency graph and invites accidental heavyweight imports. Removing it makes the
manifest reflect real usage.

## Current state

  - line 40: `"d3-shape": "catalog:frontend",`
  - line 54: `"d3": "catalog:frontend",`   ← the unused umbrella
  - line 55: `"d3-hierarchy": "catalog:frontend",`
- Verified unused: `grep -rn "from 'd3'"` across `packages/*/src` returns 0 matches;
  granular imports live in `packages/ui/src/composables/module-graph.ts` (`d3-hierarchy`, `d3-shape`).
- `d3` is declared **only** in `packages/rolldown/package.json` (not in other packages).
- The catalog entry `d3: ^7.9.0` in `pnpm-workspace.yaml` under `catalogs.frontend` can stay
  (harmless if unreferenced) — see STOP conditions; do not remove catalog entries in this plan.
